Readify User Guide

Everything you need to know about using Readify

Interface Overview

Readify's interface consists of several key components:

  • Menu Bar: Contains File and Help menus for common operations
  • Toolbar: Quick access to Open, Back, and Forward functions
  • Navigation Tree: Shows available documents in the assets folder
  • Content Area: Displays the selected HTML document
  • Status Bar: Shows information about the currently loaded file

Opening Documents

There are several ways to open HTML documents in Readify:

  1. Use the File → Open menu option
  2. Click the Open button in the toolbar
  3. Use the keyboard shortcut Ctrl+O
  4. Select a file from the navigation tree (for assets folder contents)
Tip: Readify automatically loads index.html from the assets folder when it starts up, if the file exists.

Navigation Features

The navigation tree on the left side of the interface shows all HTML files in the assets folder. Simply click on any file name to load it in the content area.

The Back and Forward buttons in the toolbar are present for future functionality but are currently disabled as specified in the design.

System Requirements

Readify is built with wxWidgets and has minimal system requirements:

  • Windows 7 or later (for Windows builds)
  • Minimal disk space (under 10MB)
  • No additional dependencies required

Building from Source

If you want to build Readify from source, you'll need:

Dependencies:

- CMake 3.16 or later

- wxWidgets library (core, base, html components)

- C++17 compatible compiler

Build steps:

cmake -S . -B build

cmake --build build --config Release
